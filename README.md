# propiedash-practice

Este es el patio de juego. Aquí los desarrolladores nuevos aprenden el flujo de git y
pull requests **sin riesgo**: no hay nada de producción, no se puede romper nada real.

Si eres nuevo: tu primera tarea está en el Paso 4 de tu guía de onboarding. En resumen,
te agregas a [`CONTRIBUIDORES.md`](./CONTRIBUIDORES.md) siguiendo un flujo de branch → commit
→ push → PR. Cuando tu PR se apruebe y se fusione, tu nombre queda en la lista.

Nada aquí afecta propiedash.com. Equivócate con confianza: de eso se trata.
