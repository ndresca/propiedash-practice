# propiedash-practice

El patio de juego del onboarding. Aquí los desarrolladores nuevos aprenden el flujo de git
y de pull requests **construyendo algo de verdad**, sin riesgo: no hay nada de producción,
no se puede romper nada real.

## Tu primer PR: tu propia tarjeta de propiedad

Propiedash es, en el fondo, propiedades. Así que tu primera tarea es **hacer una tarjeta de
propiedad** con la marca de Propiedash, y verla en tu navegador.

1. En [`ejemplo/tarjeta-ejemplo.html`](./ejemplo/tarjeta-ejemplo.html) hay una tarjeta lista,
   con los colores y el estilo de Propiedash. Ábrela en tu navegador para verla.
2. Cópiala a **`propiedades/tu-nombre.html`** (por ejemplo `propiedades/juan.html`).
3. Cámbiale los datos a una **propiedad inventada tuya**: el precio, el título, la zona, los
   cuartos, el DASHCODE, el agente. Ábrela otra vez en el navegador y ve cómo cambió.
4. Haz el flujo de git → pull request para subirla (tu guía lo corre contigo).

Cuando tu PR se apruebe y se fusione, tu tarjeta queda en la carpeta `propiedades/` junto a
las de todos. Ese es tu primer PR.

> Cada quien crea SU propio archivo en `propiedades/`, así nadie se pisa con nadie. No edites
> las tarjetas de los demás.

## Está en vivo en la web

Este repo se publica solo en internet (GitHub Pages). Portada:
**https://ndresca.github.io/propiedash-practice/**. Cuando tu PR se fusiona a `main`, tu
trabajo queda en vivo en una URL real, un **deploy**, igual que propiedash.com.

## ¿Ya terminaste? Sigue practicando

Cuando tengas tu primera tarjeta, hay una escalera de **proyectos de práctica** (una página
de resultados, la ficha de una propiedad, un perfil de agente, favoritos y filtros con
JavaScript, cambio de moneda USD/Bs). Están en [`PROYECTOS.md`](./PROYECTOS.md). Todo en el
sandbox, todo con la marca de Propiedash, cada uno un escalón más.

Nada de esto afecta propiedash.com. Equivócate con confianza: de eso se trata.
